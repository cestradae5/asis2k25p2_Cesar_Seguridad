Introducción

Este manual tiene como objetivo orientar al usuario en el uso del Sistema de Consultas SQL, el cual permite realizar consultas simples y compuestas de manera rápida y segura.

El sistema facilita la obtención de información sin necesidad de conocimientos avanzados en SQL, y a la vez ofrece la opción de crear consultas personalizadas. Además, incluye validaciones que notifican al usuario cuando una consulta está mal escrita o cuando se intenta acceder a una tabla inexistente, garantizando así un manejo correcto de la información.

Presentación

Antecedentes del sistema

En el contexto de la gestión de información, las consultas a bases de datos son fundamentales para obtener datos de manera rápida y precisa. Sin embargo, para usuarios que no dominan el lenguaje SQL, construir consultas puede resultar complejo.

El sistema de Módulo de Consultas SQL surge como una solución práctica que permite a los usuarios crear, ejecutar, editar y eliminar consultas de forma intuitiva mediante una interfaz gráfica. Este módulo combina la flexibilidad del lenguaje SQL con una interfaz sencilla que facilita tanto consultas simples como consultas más avanzadas con condiciones, uniones y ordenamientos.

Este sistema se integra dentro de un entorno académico/práctico, buscando demostrar la aplicación de herramientas de programación, diseño de interfaces y administración de bases de datos para resolver necesidades reales de interacción con la información.

Objetivos del Sistema

Objetivo General

  • Facilitar la creación, ejecución y administración de consultas SQL mediante una interfaz gráfica que no requiera conocimientos avanzados del lenguaje SQL.

Objetivos Específicos

  • Permitir la creación de consultas simples y compuestas, seleccionando tablas, campos, condiciones y ordenamientos de manera guiada.
  • Ofrecer un mecanismo para guardar y reutilizar consultas en la base de datos.
  • Proporcionar opciones para editar consultas existentes de manera sencilla y flexible.
  • Implementar funcionalidades para buscar y eliminar consultas registradas, manteniendo una gestión completa de las mismas.
  • Brindar un entorno amigable para que los usuarios puedan visualizar y exportar resultados de forma rápida y confiable.

Generalidades del Sistema

Pantalla Principal

Descripción: Pantalla de bienvenida. Desde aquí el usuario elige el tipo de consulta que quiere realizar o accede a la ayuda del sistema.

Botones

  • Consultas Simples (flecha azul) → Sirve para hacer consultas básicas seleccionando tablas y campos.
  • Consulta Compleja (flecha amarilla) → Sirve para hacer consultas avanzadas con condiciones, filtros y ordenamientos.
  • Ayuda (flecha negra) → Sirve para mostrar la guía de uso del sistema.
  • Cerrar (flecha roja) → Sirve para salir del programa.

Consultas Simples

Este módulo permite al usuario crear y ejecutar consultas básicas a la base de datos.

  • Está pensado para seleccionar directamente una tabla (Query) y obtener sus datos sin necesidad de escribir SQL manualmente.
  • El sistema genera automáticamente la sentencia en Cadena Generada.
  • El usuario puede elegir cómo ordenar los resultados (ASC o DESC) y luego ejecutar la consulta para ver la información en el área de resultados (zona gris).
  • Además, el usuario puede guardar la consulta para utilizarla posteriormente.

Botones (Consultas Simples)

  • Nombre de la tabla (desplegable) – (flecha verde): Permite elegir la tabla sobre la cual se realizará la consulta. En este caso está seleccionada empleados.
  • Nombre de empleado (campo de texto) – (flecha amarilla): Campo opcional para escribir el nombre (o parte del nombre) de un empleado y filtrar los resultados.
  • Ordenamiento ASC / DESC (opciones de radio) – (flecha naranja): Define el orden en que se mostrarán los resultados:
    • ASC → ascendente.
    • DESC → descendente.
  • Botón “Buscar” (ícono de lupa) – (flecha azul): Ejecuta la consulta con los parámetros seleccionados (tabla, nombre de empleado, orden) y muestra los resultados en la tabla de la parte inferior.
  • Tabla de resultados – (flecha morada): Muestra la información obtenida de la base de datos.
  • Botón “Consulta Compleja” – (flecha roja): Lleva al usuario al módulo de Consultas Compuestas, donde puede escribir SQL avanzado o aplicar filtros más complejos.

Consulta Compleja

Este módulo permite al usuario crear y ejecutar consultas avanzadas a la base de datos.

  • A diferencia de la consulta simple, aquí se pueden construir sentencias SQL personalizadas en la Consola de SQL.
  • Permite trabajar con múltiples condiciones, filtros y operadores lógicos.
  • Ofrece la posibilidad de ordenar resultados (ASC/DESC).
  • Incluye opciones para guardar, editar y eliminar consultas previamente definidas.
  • Muestra en el área de resultados la salida completa de la consulta ejecutada.

Validaciones y Manejo de Errores

  1. Validación de consultas SQL.

    Cuando el usuario escribe una sentencia en la Consola de SQL (Consulta Compleja) y esta contiene errores de sintaxis o comandos inválidos, el sistema:
    Detecta el error de la ejecución.
    Muestra un mensaje claro de advertencia indicando que la consulta está mal construida.
    No ejecuta la consulta hasta que el usuario la corrija.
    Ejemplo: si se escribe SELEC * FROM clientes en lugar de SELECT * FROM clientes, el sistema notificará que hay un error en la instrucción.

  2. Búsqueda de tablas inexistentes.

    En los apartados de Consultas Simples y Consultas Compuestas, cuando el usuario intenta buscar o ejecutar una consulta sobre una tabla que no existe en la base de datos:
    El sistema valida la existencia de la tabla antes de continuar.
    Si la tabla no existe, muestra un mensaje de error informativo (ej. “La tabla seleccionada no existe en la base de datos”).
    No se ejecuta la consulta para evitar resultados vacíos o caídas del sistema.

  3. Beneficios para el usuario.
    • El sistema guía al usuario en caso de equivocaciones.
    • Evita que consultas inválidas o referencias a tablas inexistentes interrumpan el flujo.
    • Los mensajes de error son claros y específicos.
